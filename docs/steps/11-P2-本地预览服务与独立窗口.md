# P2 本地预览服务与独立窗口

## 目标

实现本地预览服务和独立预览窗口，让用户可以在接近真实浏览器环境中查看当前页面。

## 范围

- 主进程启动本地 HTTP 预览服务。
- 预览当前页面导出结果。
- 支持独立预览窗口。
- 支持刷新预览。
- 支持关闭预览服务。
- 为后续二维码手机预览预留 URL。

## 不做什么

- 不实现公网分享。
- 不实现二维码 UI。
- 不实现热更新到毫秒级。
- 不实现多设备调试面板。

## 前置依赖

- 需要完成 [10-P2-HTML导出器与代码视图.md](./10-P2-HTML导出器与代码视图.md)。

## 实现要点

### 1. 预览服务位置

预览服务必须在主进程中运行：

```text
electron/services/previewService.ts
```

渲染进程只通过 preload 请求：

```ts
window.pageforge.preview.start(pageId)
window.pageforge.preview.refresh(pageId)
window.pageforge.preview.stop()
window.pageforge.preview.openWindow(pageId)
```

### 2. 内容来源

预览内容来自 HTML exporter：

- 启动或刷新时调用 exporter。
- 将生成文件写入临时预览目录，或直接由内存响应。
- 不读取画布 DOM。

### 3. 端口策略

- 默认尝试固定端口，例如 `4173`。
- 如果被占用，自动寻找可用端口。
- 返回实际 preview URL。

### 4. 独立窗口

独立预览窗口：

- 与编辑器窗口隔离。
- 不暴露 Node API。
- 不暴露编辑器 preload API。
- 加载本地 preview URL。

### 5. 生命周期

- 应用退出时关闭预览服务。
- 项目关闭时清理临时预览文件。
- 页面变更后，用户点击刷新预览即可更新。

### 6. 安全

- 预览服务只绑定本地地址，默认 `127.0.0.1`。
- 后续如支持手机扫码，需明确用户授权后绑定局域网地址。
- 预览 HTML 中的自定义脚本默认受控。

## 验收标准

- 点击预览后启动本地服务。
- 应用返回可访问的 preview URL。
- 独立预览窗口能显示当前页面。
- 修改页面后刷新预览能看到新结果。
- 关闭预览后端口释放。
- 预览窗口没有编辑器权限。
- 端口占用时能自动换端口。

## 测试建议

- previewService start/stop 测试。
- 端口占用 fallback 测试。
- exporter 集成 smoke test。
- 手动验证独立窗口权限隔离。
