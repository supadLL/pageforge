# P1 图层面板与节点管理

## 目标

实现图层树，让用户可以从结构视角管理页面节点，包括选中、重命名、显隐、锁定、删除。

## 范围

- 渲染当前页面 Node Tree。
- 支持点击图层选中节点。
- 支持重命名节点。
- 支持隐藏/显示节点。
- 支持锁定/解锁节点。
- 支持删除节点。
- 支持展开/折叠容器节点。

## 不做什么

- 不实现复杂拖拽改层级。
- 不实现批量选择。
- 不实现搜索过滤。
- 不实现右键菜单的完整能力。

## 前置依赖

- 需要完成 [05-P1-属性面板自动表单.md](./05-P1-属性面板自动表单.md)。

## 实现要点

### 1. 图层数据

图层面板直接读取当前页面 `root`，不要维护第二份树结构。

### 2. 节点显示名

优先级：

1. `node.name`
2. 组件 `label`
3. `node.type`

### 3. 状态规则

- `hidden`：画布中不显示或低透明显示，导出默认不包含隐藏节点。
- `locked`：画布中不可选中、不可拖拽，但图层面板仍可选中。
- `PageRoot` 不允许删除。

### 4. 删除规则

删除节点时：

- 同时删除其所有子节点。
- 如果删除的是当前选中节点，选中其父节点。
- 删除前可先不做确认弹窗，后续由撤销能力兜底。

### 5. 展开状态

展开/折叠属于编辑器 UI 状态，不写入 Project。

## 验收标准

- 图层面板能展示完整 Node Tree。
- 点击图层能选中对应画布节点。
- 画布选中变化能同步高亮图层。
- 可以重命名节点。
- 可以隐藏/显示节点。
- 可以锁定/解锁节点。
- 可以删除普通节点。
- PageRoot 不能被删除。

## 测试建议

- Node Tree 遍历工具测试。
- 删除节点工具测试。
- 查找父节点工具测试。
- hidden/locked 状态更新测试。
