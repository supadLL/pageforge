# P4 AI 微调

## 目标

让用户用自然语言指令修改页面（"把所有按钮改成圆角红色"、"顶部加个带搜索框的导航栏"），AI 返回结构化 Patch 列表，经校验后应用，且可撤销。支持选中范围 AI 改（只对选中节点生效）。

## 范围

- 定义 `AiPatchSet` 与 `AiPatch` 协议（`docs/01 §9` 已有草案，本步定稿）。
- 实现 `validateAiPatchSet` / `normalizeAiPatchSet`。
- 实现 `applyPatchCommand`：把 patch 列表打包成一条可撤销命令。
- 主进程 `aiService.editByPrompt`：把当前选中子树 + 指令发给 AI，返回 patch 列表。
- 渲染进程"AI 微调"输入框：在工具栏或属性面板，输入指令 → 应用。
- 选中范围 AI 改：框选/选中节点时，prompt 附带选中子树，patch 的 nodeId 限定在选中范围内。
- AI 改动可撤销（一条历史记录）。

## 不做什么

- 不做 AI 改动的前后 diff 可视化（MVP 直接应用 + 撤销兜底）。
- 不做多轮对话上下文（每次微调独立请求）。
- 不做 patch 的部分应用（要么全应用要么全拒绝）。
- 不做自然语言到复杂动画/交互的转换（MVP 只覆盖样式/结构/文案）。

## 实现要点

### 1. Patch 协议定稿

```ts
// packages/schema/src/ai/patch.ts
export interface AiPatchSet {
  summary: string
  patches: AiPatch[]
}

export type AiPatch =
  | { op: 'addNode'; parentId: string; index?: number; node: Partial<Node> }
  | { op: 'removeNode'; nodeId: string }
  | { op: 'moveNode'; nodeId: string; parentId: string; index?: number }
  | { op: 'updateProps'; nodeId: string; props: Record<string, unknown> }
  | { op: 'updateStyle'; nodeId: string; style: Partial<StyleMap>; breakpoint?: BreakpointName }
  | { op: 'renameNode'; nodeId: string; name: string }

export function validateAiPatchSet(input: unknown): ValidationResult<AiPatchSet>
export function normalizeAiPatchSet(set: AiPatchSet, root: Node): { patches: AiPatch[]; warnings: string[] }
```

`normalizeAiPatchSet` 流程：
1. 校验 patch 结构。
2. 对每个 patch 检查目标 nodeId 是否存在于当前树（addNode 检查 parentId）。
3. 对 `addNode` 的 `node` 补 id、补默认 props/style（复用 `createNode`）。
4. 对 `updateStyle` 的 style key 校验是否在白名单。
5. 丢弃非法 patch 并记 warning。

### 2. applyPatchCommand

新增命令类型 `applyPatch`（Step 07 已预留 type，本步实现 executor）：

```ts
interface ApplyPatchPayload {
  patches: AiPatch[]
  /** 应用前的 root 快照（用于 revert） */
  before: Node
}
```

- apply：按顺序对 root 应用每个 patch（复用 treeOps）。
- revert：用 `before` 快照直接替换 root。
- 一条 `applyPatch` 命令 = 一条历史记录，无论内部多少 patch。

### 3. 主进程 editByPrompt

```text
aiService.editByPrompt(root | selectedSubtree, prompt: string): Promise<AiPatchSet>
```

流程：
1. 序列化当前 root（或选中子树）为紧凑 JSON（只保留 id/type/props/style/children，去掉 events/responsive 减小 token）。
2. system prompt：声明输出必须是 `AiPatchSet` JSON，给出 patch op 枚举与示例。
3. user message：序列化的树 + 用户指令。
4. 调用 `provider.chat`。
5. 提取 JSON，返回 `AiPatchSet`。

### 4. 选中范围

- 若 `editor.selectedNodeId` 非空，prompt 中明确"只修改选中节点及其后代"，并把 patch 的 nodeId 限定在选中子树内（normalize 时校验）。
- 未选中时，对整页生效。

### 5. 渲染进程入口

- 工具栏加"AI 微调"输入框（或快捷键 Ctrl+I 聚焦）。
- 输入指令回车 → 调 `window.pageforge.ai.editByPrompt`。
- 加载态 → 返回后直接应用（applyPatchCommand）并 selectNode 到第一个被改的节点。
- 失败时 toast 提示，不改动画布。
- 可撤销（Ctrl+Z 回到微调前）。

### 6. Prompt 关键约束

system prompt 包含：
- patch op 枚举与字段说明。
- "只返回 JSON"。
- style key 白名单。
- "nodeId 必须来自我提供的树"。
- 一个"把按钮改红色"的最小示例。

## 验收标准

- 选中一个 Button，输入"改成红色圆角"，AI 返回 `updateStyle` patch，应用后按钮变红圆角，可撤销。
- 未选中时输入"顶部加一个标题"，AI 返回 `addNode` patch，应用后顶部出现 Heading。
- AI 返回引用不存在的 nodeId 时，该 patch 被丢弃并记 warning，其他 patch 仍应用。
- 一次微调只产生一条历史记录。
- 非法 JSON 返回时画布不变，toast 提示。
- 选中范围模式下，patch 不会改到选中子树外的节点。
