# P1 项目保存打开与资源导入

## 目标

实现 `.pageforge` 工程目录的保存、打开和图片资源导入，让编辑器状态可以跨会话恢复。

## 范围

- 定义项目目录结构。
- 保存 `project.json`。
- 打开已有项目。
- 导入图片到 `assets/`。
- 图片组件引用 `assetId`。
- 记录最近打开项目。
- dirty 状态与保存联动。

## 不做什么

- 不实现云同步。
- 不实现版本快照 UI。
- 不实现多人协作。
- 不实现复杂资源管理器。

## 前置依赖

- 需要完成 [08-P1-拖拽添加与节点重排.md](./08-P1-拖拽添加与节点重排.md)。

## 实现要点

### 1. 工程目录

```text
my-project.pageforge/
├── project.json
├── assets/
├── snapshots/
└── meta.json
```

MVP 必须写入：

- `project.json`
- `assets/`
- `meta.json`

`snapshots/` 可先创建目录但不实现 UI。

### 2. 主进程服务

建议：

```text
electron/services/projectService.ts
electron/services/assetService.ts
```

所有文件操作必须在主进程。

### 3. preload API

```ts
window.pageforge.project.create()
window.pageforge.project.open()
window.pageforge.project.save(project)
window.pageforge.asset.importImage()
```

参数和返回值必须类型化。

### 4. 原子保存

保存 `project.json` 时：

1. 写入 `project.json.tmp`。
2. 校验写入结果。
3. 替换 `project.json`。

### 5. 资源导入

导入图片时：

- 复制到项目 `assets/`。
- 生成 `Asset` 记录。
- 返回 `assetId`。
- Image 节点 props 使用 `assetId`。

### 6. 打开项目

打开时：

- 读取 `project.json`。
- 校验 schema。
- 如版本过旧，走迁移入口。
- 失败时展示明确错误，不覆盖原文件。

## 验收标准

- 可以新建项目目录。
- 可以保存当前 Project。
- 关闭应用后重新打开项目，页面恢复。
- 导入图片后，图片复制到 `assets/`。
- Image 节点可以显示导入图片。
- 保存成功后 dirty 状态清零。
- 打开非法项目时不会崩溃，并提示校验错误。
