# P1 命令系统与撤销重做

## 目标

把编辑器修改统一收敛到 Command 系统，并实现撤销/重做，为后续 AI Patch、拖拽、保存前变更检测打基础。

## 范围

- 定义 Command 数据结构。
- 实现 command dispatcher。
- 实现 history store。
- 接管新增、删除、属性修改、样式修改、重命名、显隐、锁定。
- 实现撤销和重做。

## 不做什么

- 不实现 AI Patch。
- 不实现协同编辑。
- 不实现跨会话历史恢复。
- 不实现复杂 diff 可视化。

## 前置依赖

- 需要完成 [06-P1-图层面板与节点管理.md](./06-P1-图层面板与节点管理.md)。

## 实现要点

### 1. Command 结构

推荐使用可序列化 command，而不是把函数直接放进历史栈：

```ts
interface EditorCommand {
  id: string
  type: string
  label: string
  createdAt: string
  payload: unknown
  before?: unknown
  after?: unknown
}
```

然后由 command handlers 实现 apply/revert。

### 2. 必须支持的命令

- `addNode`
- `removeNode`
- `updateProps`
- `updateStyle`
- `updateResponsiveStyle`
- `renameNode`
- `setNodeState`

`moveNode` 和 `reorderNode` 可在第 08 步补齐，也可以本步先实现。

### 3. 历史栈

```ts
interface HistoryState {
  undoStack: EditorCommand[]
  redoStack: EditorCommand[]
  dirty: boolean
}
```

规则：

- 执行新 command 后清空 redoStack。
- undo 后 command 进入 redoStack。
- redo 后 command 回到 undoStack。
- 保存项目后 `dirty = false`。

### 4. 合并策略

属性连续输入需要合并：

- 同一节点
- 同一字段
- 短时间窗口内，例如 500ms

MVP 可以先做简单 debounce，也可以先不做高级合并，但要避免每个键盘输入都产生难以使用的历史。

### 5. UI 入口

- 工具栏按钮：撤销、重做。
- 快捷键：Ctrl+Z、Ctrl+Y / Ctrl+Shift+Z。

## 验收标准

- 新增节点可以撤销/重做。
- 删除节点可以撤销/重做。
- props 修改可以撤销/重做。
- style 修改可以撤销/重做。
- 图层重命名可以撤销/重做。
- 显隐/锁定可以撤销/重做。
- 执行新命令后 redo 栈清空。
- dirty 状态能反映是否有未保存修改。

## 测试建议

- 每个 command 的 apply/revert 单元测试。
- undo/redo 栈行为测试。
- dirty 状态测试。
- 连续属性修改合并测试。
