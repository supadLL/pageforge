# P0 组件注册表与默认组件

## 目标

建立组件注册表，让编辑器可以通过统一协议知道组件的默认 props、默认 style、属性 schema、是否允许子节点。

## 范围

- 定义 `ComponentDefinition`。
- 建立组件注册表。
- 实现 MVP 默认组件定义。
- 提供创建组件节点的 factory。

## 不做什么

- 不实现复杂组件 UI。
- 不实现拖拽排序。
- 不实现属性面板完整交互。
- 不实现框架导出。

## 前置依赖

- 需要完成 [02-P0-核心Schema与校验基础.md](./02-P0-核心Schema与校验基础.md)。

## 实现要点

### 1. 组件定义

```ts
interface ComponentDefinition {
  type: ComponentType
  label: string
  category: ComponentCategory
  icon?: string
  defaultProps: Record<string, unknown>
  defaultStyle: StyleMap
  propSchema: JsonSchema
  acceptsChildren: boolean
  allowedChildren?: ComponentType[]
}
```

### 2. MVP 组件

必须包含：

- `PageRoot`
- `Container`
- `Card`
- `Heading`
- `Text`
- `Button`
- `Image`
- `Input`
- `Divider`

### 3. 默认行为

- `PageRoot`、`Container`、`Card` 允许 children。
- 其他组件第一版不允许 children。
- 每个组件必须有 `defaultProps` 和 `defaultStyle`。
- 每个组件必须有 `propSchema`。

### 4. 创建节点

提供：

```ts
createNode(type: ComponentType, overrides?: Partial<Node>): Node
```

该方法负责：

- 生成 id。
- 合并默认 props。
- 合并默认 style。
- 根据 `acceptsChildren` 决定是否初始化 `children`。

## 验收标准

- 所有 MVP 组件都能通过 `createNode` 创建合法节点。
- 非法组件类型创建失败并返回明确错误。
- 非容器组件不会带 children。
- 容器组件默认带空 children。
- 组件 `propSchema` 能校验默认 props。

## 测试建议

- 每个默认组件创建测试。
- `acceptsChildren` 行为测试。
- `defaultProps` 通过 schema 测试。
- 非法组件类型测试。
