# P3-2 AI 图生页面

## 目标

实现上传截图/设计稿/草图，由 Vision AI 识别布局与样式，生成结构化 Node Tree 草稿，经过校验和归一化后加载进画布，可继续拖拽精调。

## 范围

- 定义 `GeneratedPageDraft` 与 `normalizeGeneratedDraft` / `validateGeneratedDraft`。
- 实现 `insertGeneratedDraft`：把草稿替换或合并到当前页面 root。
- 渲染进程"图生页面"入口：上传图片、调用 AI、显示加载状态、加载结果进画布。
- 主进程 `aiService.generateFromImage`：把图片转 base64、构造 prompt、调用 Vision Provider、返回草稿。
- Prompt 工程：要求模型直接返回 Node Tree JSON（带 schema 约束），失败时兜底返回 HTML 再解析。
- 草稿预览面板：在加载进画布前展示 AI 返回的结构摘要与 warnings。

## 不做什么

- 不追求复杂设计稿的一次性高保真还原（MVP 范围：单区块/单卡片级识别，整页靠拼装）。
- 不做多图拼接自动布局推断。
- 不做 AI 返回的 token 流式展示（一次性返回）。
- 不做草稿 diff 可视化（Step 14 微调才需要）。
- 不实现 HTML 解析器的完整 CSS 覆盖（只兜底常用属性）。

## 前置依赖

- 需要完成 [12-P3-AI-Provider抽象层与配置.md](./12-P3-AI-Provider抽象层与配置.md)。
- `packages/schema` 的 `validateNode` 可用于校验草稿。
- `createNode` factory 可用于补默认值。

## 实现要点

### 1. 草稿协议

```ts
// packages/schema/src/ai/draft.ts
export interface GeneratedPageDraft {
  summary: string
  tokens?: Partial<DesignTokens>
  root: Partial<Node>
  warnings?: string[]
}

export function normalizeGeneratedDraft(draft: GeneratedPageDraft): { root: Node; warnings: string[] }
export function validateGeneratedDraft(draft: unknown): ValidationResult<GeneratedPageDraft>
```

`normalizeGeneratedDraft` 流程：
1. 校验 draft 结构。
2. 递归遍历 `draft.root`，对每个节点：
   - 若 `type` 不在注册表，记 warning 并跳过。
   - 用 `createNode(type)` 取默认 props/style，再用 draft 值覆盖。
   - 补 id（`genNodeId`）。
   - 非容器组件丢弃 children。
3. 若 `root.type` 不是 `PageRoot`，包一层 PageRoot。
4. 合并 `draft.tokens` 到当前项目 tokens（不覆盖已有 key）。

### 2. 主进程 generateFromImage

```text
electron/services/aiService.ts -> generateFromImage(imagePath | buffer, opts): Promise<GeneratedPageDraft>
```

流程：
1. 读图为 base64 data URL。
2. 构造 system prompt：声明输出必须是 `GeneratedPageDraft` JSON，给出组件类型枚举与 style key 白名单。
3. 构造 user message：图片 + "请识别此页面并生成 Node Tree"。
4. 调用 `provider.chat({ messages, jsonSchemaHint })`。
5. 从 content 中提取 JSON（兼容 ```json 围栏）。
6. 解析为 `GeneratedPageDraft` 返回；解析失败返回 `{ summary, root: { type: 'PageRoot', children: [] }, warnings: ['解析失败'] }`。

### 3. Prompt 关键约束

system prompt 必须包含：
- 允许的 `type` 枚举（来自 `MVP_COMPONENT_TYPES`）。
- 允许的 style key（来自 `ALLOWED_STYLE_KEYS`）。
- token 引用格式 `$colors.primary`。
- "只返回 JSON，不要解释"。
- 一个最小示例（PageRoot + Heading + Text）。

### 4. 渲染进程入口

- 工具栏或独立面板加"图生页面"按钮。
- 点击 → 弹出图片选择（复用 `asset.chooseAndImport` 或单独文件选择）。
- 选择后显示加载态，调 `window.pageforge.ai.generateFromImage`。
- 返回后打开草稿预览面板：
  - 显示 `summary`、节点数、warnings 列表。
  - "加载到画布"按钮：调 `insertGeneratedDraft`。
  - "丢弃"按钮。
- `insertGeneratedDraft` 走 command 系统（一条 `applyPatch` 或新增 `replacePageRoot` 命令），可撤销。

### 5. 合并策略

MVP 提供两种模式（用户在草稿面板选）：
- **替换**：用草稿 root 替换当前页面 root（整页重建）。
- **追加**：把草稿 root 的 children 追加到当前页面 root 的 children 末尾（拼装）。

两种都生成可撤销的命令。

### 6. 错误处理

- 网络错误：显示明确提示，不破坏当前画布。
- AI 返回非法 JSON：保留原始响应摘要到 warnings，root 给空 PageRoot。
- 校验失败节点：warning 记录路径，跳过该节点。

## 验收标准

- 上传一张简单卡片/落地页截图，能返回合法 Node Tree 草稿。
- 草稿加载进画布后可继续拖拽、改属性、撤销。
- AI 返回非法结构时画布不崩溃，warnings 可见。
- 替换模式下，撤销可回到加载前的页面。
- 追加模式下，原页面内容保留，草稿节点追加在末尾。
- 未配置 API Key 时点击"图生页面"给出明确引导。

## 测试建议

- `normalizeGeneratedDraft` 单元测试：缺字段补默认、非法 type 跳过、非 PageRoot 包一层、token 合并。
- `validateGeneratedDraft` 测试：合法/非法结构。
- prompt 构造测试：包含组件枚举与 style 白名单。
- JSON 提取测试：带围栏、不带围栏、前后有解释文字。
- mock Provider 的端到端测试：从图片输入到草稿加载入 store。
- 草稿预览面板组件测试：summary/warnings 展示、加载/丢弃按钮。
