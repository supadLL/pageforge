# P0 工程初始化与运行骨架

## 目标

建立 PageForge 的最小可运行工程，让 Electron 主进程、preload、Vue 渲染进程可以启动并通信。

## 范围

- 初始化 `package.json`。
- 配置 Electron + Vite + Vue 3 + TypeScript。
- 建立 `electron/main`、`electron/preload`、`src` 基础目录。
- 建立最小窗口和渲染页面。
- 建立基础开发命令、构建命令和测试命令。

## 不做什么

- 不实现编辑器功能。
- 不实现项目保存。
- 不接入 AI。
- 不做复杂 UI 设计。

## 前置依赖

- Node.js 环境可用。
- 明确使用 npm、pnpm 或 yarn。建议优先使用项目初始化时选择的一种，不混用。

## 实现要点

### 1. package scripts

建议至少提供：

```json
{
  "scripts": {
    "dev": "启动 Electron + Vite 开发环境",
    "build": "构建渲染进程和主进程",
    "test": "运行单元测试",
    "typecheck": "运行 TypeScript 类型检查",
    "format": "运行格式化"
  }
}
```

### 2. Electron 安全默认值

BrowserWindow 必须启用：

- `contextIsolation: true`
- `nodeIntegration: false`
- `sandbox: true`，如遇依赖问题再单独评估
- 只加载 preload 暴露的白名单 API

### 3. preload 最小 API

第一步只暴露健康检查接口：

```ts
window.pageforge = {
  app: {
    ping(): Promise<string>
  }
}
```

### 4. 渲染进程

Vue 页面只需要证明三件事：

- 应用能显示。
- 能调用 `window.pageforge.app.ping()`。
- 能展示主进程返回结果。

## 验收标准

- `npm run dev` 可以启动桌面窗口。
- 窗口中显示 PageForge 基础页面。
- 渲染进程能通过 preload 调用主进程 ping。
- DevTools 中没有明显启动错误。
- `npm run typecheck` 通过。
- `npm run build` 至少能完成当前骨架构建。

## 测试建议

- 为 preload 暴露类型添加 TypeScript 声明。
- 对主进程 ping handler 写一个轻量单元测试或 smoke test。

## PR Checklist

- [ ] Electron 主窗口可启动。
- [ ] preload API 有类型声明。
- [ ] 渲染进程没有直接使用 Node API。
- [ ] scripts 可运行。
- [ ] README 或开发说明中写明启动命令。
