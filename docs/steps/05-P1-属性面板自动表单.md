# P1 属性面板自动表单

## 目标

实现基于组件 `propSchema` 和通用 style schema 的属性面板，让用户能编辑选中节点的内容和常用样式。

## 范围

- 显示当前选中节点信息。
- 根据 `propSchema` 自动生成 props 表单。
- 实现常用 style 编辑控件。
- 支持基础样式和当前断点覆盖样式。
- 修改后更新 Node Tree。

## 不做什么

- 不做完整设计系统编辑器。
- 不做复杂响应式可视化。
- 不做 AI 修改。
- 不做组件自定义 schema 插件。

## 前置依赖

- 需要完成 [04-P0-Node树渲染器与选中态.md](./04-P0-Node树渲染器与选中态.md)。

## 实现要点

### 1. 面板结构

建议分区：

- 基本信息：类型、名称、id。
- 内容属性：来自 `propSchema`。
- 布局：display、flexDirection、alignItems、justifyContent、gap、width、height。
- 外观：color、backgroundColor、borderRadius、boxShadow、opacity。
- 文字：fontSize、fontWeight、lineHeight、textAlign。
- 间距：padding、margin。

### 2. 表单控件映射

根据 schema 类型映射控件：

| schema | 控件 |
|---|---|
| string | 文本输入 |
| number | 数字输入 |
| boolean | 开关 |
| enum | 下拉或分段控件 |
| color 格式 | 颜色选择 |
| asset 引用 | 资源选择 |

### 3. 响应式编辑

- 默认编辑基础样式 `node.style`。
- 当用户切换到断点模式时，写入 `node.responsive[currentBreakpoint].style`。
- UI 需要显示当前字段是否来自断点覆盖。

### 4. 更新方式

该步骤可先直接调用 store 更新。  
但代码结构要预留给后续 Command 系统接管。

建议统一封装：

```ts
updateSelectedNodeProps(partialProps)
updateSelectedNodeStyle(partialStyle, breakpoint?)
```

后续第 07 步将内部替换为 command。

## 验收标准

- 选中 Text 后可以修改文本内容。
- 选中 Heading 后可以修改标题级别和文本。
- 选中 Button 后可以修改按钮文案和 variant。
- 可以修改颜色、背景、字号、圆角、padding、margin。
- 可以修改 Container/Card 的 flex 方向和 gap。
- 修改后画布立即更新。
- 切换断点后可以写入断点覆盖样式。

## 测试建议

- schema 到控件映射单元测试。
- style 更新工具函数测试。
- 断点覆盖合并逻辑测试。
