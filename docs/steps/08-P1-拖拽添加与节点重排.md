# P1 拖拽添加与节点重排

## 目标

实现从组件库拖拽添加节点，以及在容器内重排节点，让页面搭建从“点击添加”升级为基础可视化编辑。

## 范围

- 组件库拖拽到画布。
- 容器可放置高亮。
- 计算插入位置。
- 同一容器内节点重排。
- 跨容器移动节点。
- 通过 Command 记录拖拽结果。

## 不做什么

- 不实现自由画布 absolute 拖动。
- 不实现多选拖拽。
- 不实现复杂吸附线和标尺。
- 不实现成组/解组。

## 前置依赖

- 需要完成 [07-P1-命令系统与撤销重做.md](./07-P1-命令系统与撤销重做.md)。

## 实现要点

### 1. 拖拽来源

两类来源：

- 组件库：创建新节点。
- 画布节点：移动已有节点。

### 2. 放置目标

只有 `acceptsChildren = true` 的组件可以作为放置目标。

如果鼠标位于非容器节点上：

- 优先使用其父容器作为放置目标。
- 根据该节点位置计算前后插入。

### 3. 插入位置

Flex 流式布局下，MVP 可用简单策略：

- 纵向容器：根据鼠标 Y 坐标判断插入 index。
- 横向容器：根据鼠标 X 坐标判断插入 index。

### 4. Command

需要补齐：

- `moveNode`
- `reorderNode`

拖拽过程中只更新 hover 状态。  
drop 成功后才提交 command。

### 5. 视觉反馈

- 可放置容器高亮。
- 插入位置显示一条 indicator。
- 不可放置时显示禁用状态。

## 验收标准

- 可以从组件库拖 Text/Button/Image/Container 到 PageRoot。
- 可以拖到 Container/Card 内部。
- 可以在同一容器内改变节点顺序。
- 可以把节点从一个容器移动到另一个容器。
- 非容器组件不能作为放置目标。
- 拖拽完成后可撤销/重做。
- 拖拽过程中不会污染历史栈。

## 测试建议

- 计算 drop target 的工具函数测试。
- 计算 insert index 的工具函数测试。
- `moveNode` 和 `reorderNode` command 测试。
- 手动走通拖拽主流程。
